## Project Overview

Interview Assistant（面试助手）is a Markdown Q&A review tool with zero dependencies, no build step, and no framework. It has three components sharing a single core parser.

## Repository Layout

```
core/parser.js          # Pure-function Markdown Q&A parser (also runs in Node.js for tests)
core/parser.test.js     # 49 tests using Node's built-in assert
web/index.html          # Single-file web app: HTML + CSS + JS, ~1500 lines
extension/              # Chrome/Edge extension (Manifest V3)
  content-script.js     # Injects interactive Q&A cards into .md pages (GitHub, local files)
  content-style.css     # Minimal decorative styles for injected UI
  parser.js             # Copy of core/parser.js for extension use
sample-notes.md         # Demo markdown showing all supported formats
```

## Key Commands

```bash
# Run parser tests
node core/parser.test.js

# Deploy web app to GitHub Pages
git checkout gh-pages && git checkout master -- web/index.html && git commit -m "deploy" && git push && git checkout master
```

The web app is deployed by copying `web/index.html` to the `gh-pages` branch root.

## Architecture

### Parser (`core/parser.js`)

`parseMarkdown(text)` returns `{ pairs, preamble, postamble }`. Each pair: `{ id, question, answer, type, level }`.

Normalization pipeline (in order):
1. Normalize heading prefixes: `Q:` / `Question:` / `问题：` / numbered `Q1:` → standard `Q:`
2. Detect numbered topics (`### 1. topic`, `### 2、topic`) as implicit questions
3. Fallback: any `###+` heading not matching exclusion keywords becomes a question
4. `A:` headings mark explicit answer boundaries; content between Q and next heading is implicit answer

The same parser is duplicated in three places. Changes to parsing logic must be synced across `core/parser.js`, `web/index.html` (inline), and `extension/parser.js`.

### Web App (`web/index.html`)

A single file containing all HTML, CSS, and JavaScript. No external dependencies.

**Views** (managed by `switchView()`):
- `edit` — `<textarea>` for Markdown input, file upload via button or drag-and-drop
- `review` — accordion cards with toggle icons, generated by `renderReview()` from `parseResult`
- `error` — error collection bookmarks, same card pattern as review

**State**: Two global variables — `parseResult` (parser output) and `currentView` (string). UI state (which cards are revealed) is read from the DOM via `querySelectorAll('.qa-card.revealed')`.

**Persistence** (`localStorage`):
- `ia-markdown` — raw markdown text (debounced 500ms save)
- `ia-theme` — `'light'` or `'dark'`
- `ia-error-collection` — JSON array of `{ id, question, answer, source, sourceId, createdAt }`

**Styling**: CSS custom properties on `:root` and `[data-theme="dark"]`. Animations use CSS transitions (`max-height` for accordion, `transform` for toggle icon rotation). No CSS framework.

**Tracking**: Google Analytics 4 (`G-7K6V8ESG2L`) via `track()` function for view switches, toggles, theme changes, and shortcut usage.

### Extension (`extension/`)

Three detection modes in `content-script.js`:
1. **Raw text mode** — page is raw .md content; replaces entire body with interactive Q&A cards
2. **DOM light mode** — GitHub/GitLab pages; modifies existing DOM to hide answers, add toggle buttons
3. **Fallback mode** — injects a floating button that opens a Blob URL with standalone review page

Uses inline styles (via `style` attribute) to prevent host page CSS from overriding the extension UI.

## Patterns When Editing

- **The parser is duplicated** — any fix to `core/parser.js` must also be applied to the inline copy in `web/index.html` and `extension/parser.js`
- **CSS uses custom properties exclusively** — always use `var(--bg)`, `var(--border)`, `var(--accent)`, etc., never hardcode colors
- **Event handlers on generated HTML** use inline `onclick` with `event.stopPropagation()` for nested clickable elements
- **Keyboard shortcuts** use `Shift+<key>` to avoid conflicts with browser defaults and text editing
- **Emoji-based icons** — the project uses emoji (👁️, 🙈, ✏️, 📌, ❌) rather than icon fonts or SVGs
